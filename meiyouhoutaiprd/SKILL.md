# 美柚中后台 PRD 生成器

## 概述

本 skill 基于美柚 TAPD 中后台需求模板（模板ID: 中后台需求模版），按标准三段式结构生成需求文档：

- **需求背景（20%）**：业务大背景 → 业务子背景 → 现状判断及问题
- **项目目标（5%）**：目标描述 + 迭代节奏 + 风险预判
- **需求方案（80%）**：名词定义 → E-R图 → 结构图 → 流程图 → 原型图 → 需求说明 → 协同需求

输出格式：**直接生成 HTML**（含 Mermaid 图表自动渲染为 PNG 并嵌入）。也可生成 .docx（用 scripts/generate_meiyou_prd.py）。

## 文档结构

完整模板见 references/template.md，核心章节：

### 标题格式
{角色（运营/产品）}可以通过{功能}获得{价值}

### 一、需求背景（20%）
| 章节 | 占比 | 内容 |
|------|------|------|
| 1.1 业务大背景 | 5% | 对齐团队 OKR，说明需求服务于哪个大 O |
| 1.2 业务子背景 | 5% | 用户反馈/Case 分析，商业化需求则论述客户信息 |
| 1.3 现状判断及问题 | 10% | 四列表格：现状 → 问题判断 → 历史需求 → 解决方案 |

### 二、项目目标（5%）
| 章节 | 内容 |
|------|------|
| 2.1 目标描述 | 本次迭代价值（运营灵活配置/提升人效/降低成本） |
| 2.2 迭代节奏（非必填） | 分阶段实现计划 |
| 2.3 风险预判（非必填） | 技术难点、延期风险、影响范围 |

### 三、需求方案（80%）
| 章节 | 占比 | 内容 |
|------|------|------|
| 3.1 名词定义 | 非必填 | 名词/定义 对照表 |
| 3.2 E-R图 | 非必填 | 实体关系图（必须使用中文命名实体、属性和关系，如"广告分组"、"包含"、"广告源"，不得使用英文） |
| 3.3 产品结构图 | 10% | 模块/功能脑图 |
| 3.4 产品流程图 | 10% | 多角色泳道图 |
| 3.5 原型图 | 非必填 | 后台原型 |
| 3.6 需求说明 | 60% | 功能模块/功能点/优先级 + 详细说明 |
| 3.7 协同方需求 | 非必填 | 运营/编辑/商务侧配合内容 |

### 元数据字段
- 状态、父需求、分类、业务、迭代、处理人、优先级、需求类型、需求难度、技术等级、预估工时等

## 工作流

### Step 1：确认基本信息
向用户确认：
- **需求标题**：按格式 \"XX可以通过XX获得XX\"
- **创建模板**：默认\"中后台需求模版\"
- **分类/业务**：如 会员、广告、内容
- **迭代**：所属迭代版本
- **优先级**：High/Middle/Low
- **需求类型**：基础建设/新功能/优化
- **需求难度**：A 全新能力 / B 现有能力迭代 / C 简单配置
- **处理人**：默认当前用户

### Step 2：填充需求内容
按章节引导用户填写：
1. 需求背景（必须）
2. 项目目标（必须）
3. 需求方案（核心，重点填写 3.3-3.6）

### Step 3：直接生成 HTML
按美柚中后台 PRD 模板直接生成 HTML 文件，流程如下：
1. 按模板填充内容，输出为 PRD Markdown 文件
2. 调用 scripts/render_mermaid_images.py 将 Mermaid 图表渲染为 PNG
3. 调用 scripts/md_to_tapd_html.py 将 Markdown 转为 HTML
4. 将渲染好的 PNG 图片嵌入 HTML 的占位符中

也可输出 .docx：调用 scripts/generate_meiyou_prd.py

## 自动填充 TAPD（可选）

生成 PRD 后，可自动将内容填充到指定的 TAPD 需求单。

### 前置条件
- 用户已在 Chrome 中登录 TAPD（tapd.meiyou.com）
- 目标需求单已存在（先手动创建空白需求单，选择\"中后台需求模版\"）

### 填充流程

1. 生成结构化 JSON 数据（同 docx 生成的数据格式）
2. 调用 scripts/fill_tapd.mjs：
   `ash
   node scripts/fill_tapd.mjs \\\\
     --story-id <需求ID> \\\\
     --workspace <工作空间ID> \\\\
     --data <json文件路径>
   `
3. 脚本自动打开 Chrome（复用登录态），导航到编辑页
4. 自动填充：标题、描述（含表格）、处理人、优先级、难度等
5. 填充完成后手动检查并点击【提交】

### TAPD URL 解析
从 TAPD 链接提取参数：
- https://www.tapd.meiyou.com/tapd_fe/{workspace_id}/story/detail/{story_id}
- 示例：workspace_id=21039721, story_id=1121039721001508584
## HTML 生成流程（推荐）

直接生成完整 HTML，无需中间转换步骤。

### 一键生成流程

1. **填充 PRD 内容**：按美柚中后台模板填充所有章节，生成 PRD Markdown 文件（Mermaid 代码块包含在 MD 中）
2. **渲染 Mermaid 图表为 PNG**：
   ```bash
   python scripts/render_mermaid_images.py --input <prd.md> --output-dir <output_dir>/diagrams/
   ```
   - 自动识别 4 种图表类型（E-R图、产品结构图、流程图、状态流转图）
   - 使用 Playwright 无头浏览器 + Mermaid CDN 渲染
   - 输出 PNG 到 diagrams/ 目录
3. **将 Markdown 转为 HTML**：
   ```bash
   python scripts/md_to_tapd_html.py --input <prd.md> --output <prd.html>
   ```
4. **嵌入 PNG 图片**：将 diagrams/ 目录下的 PNG 替换 HTML 中的黄色虚线占位符，嵌入为 `<img>` 标签

### HTML 特性
- 表格：border-collapse 紧凑排版
- Mermaid 图表：自动渲染为 PNG 并嵌入
- 标题、列表、引用块、粗体完整保留
- 相对路径引用，同目录下可直接打开查看

### 粘贴到 TAPD
- 浏览器打开生成的 .html → 全选复制
- TAPD 编辑页粘贴 (Ctrl+V)

### Step 4：确认修改
生成后请用户确认，支持迭代修改。
