# DESIGN.md

## 气质与意象
企业级后台管理系统——如同一间高效运转的运营指挥中心，数据密度高、操作路径短、信息层级清晰。视觉上追求"冷静的秩序感"，让运营人员在密集配置中仍能快速定位、精准操作。

## 配色方案
- 主色：深青灰（Slate 900），传达专业与可靠
- 强调色：玫瑰红（Rose 500），用于关键操作与状态高亮，源自"促销/活动"的商业活力
- 辅助色：琥珀黄（Amber 500），用于警告与待处理状态
- 成功色：翠绿（Emerald 500），用于完成与上线状态
- 中性色阶：Slate 50-800，覆盖背景、边框、文字的全场景

## 字体排版
- 字体族：系统默认中文字体栈（PingFang SC / Microsoft YaHei / system-ui）
- 标题：font-semibold，视觉锚点
- 正文：font-normal，保证长表单阅读舒适
- 数据/标签：font-mono，数字对齐与等宽

## 页面结构
- 左侧固定侧边栏（240px），承载全局导航
- 顶部面包屑 + 操作区
- 内容区白底卡片，圆角 8px，轻阴影
- 表格行密度适中，悬停高亮

## 组件规范
- 按钮：主操作用实心玫瑰红，次操作用描边，危险操作用红色系
- 状态标签：圆角药丸样式（Badge），颜色映射语义（草稿=灰、进行中=绿、已结束=灰暗）
- 表单：Label 在上，输入框满宽，错误提示红色行内
- 步骤向导：横向步骤条，当前步骤高亮，已完成步骤打勾

## 动效与交互
- 页面切换：无过度动效，保持后台效率优先
- 模态框：fade-in + scale 微动
- 表格排序：即时响应
- 按钮点击：scale(0.97) 微反馈

## 设计禁忌
- 禁止大面积渐变背景——后台不是营销页面
- 禁止动画图标或闪烁元素——干扰信息获取
- 禁止纯装饰性插图——每一像素都应有信息价值
- 禁止超过 3 级的弹窗嵌套
